Corpus-Aware Weight Optimization

Weights are dynamically computed by evaluating each method on corpus samples

The 4 Similarity Methods
Semantic
Embedding-based cosine similarity using sentence embeddings (384-dim)
Field Score
Weighted per-field TF-IDF scoring — matches in Idea Name count more than Description or Domain
BM25
Probabilistic relevance framework, balances term frequency and document length
N-Gram (Bigram)
Sequence-based phrase overlap using 2-grams, captures phrase-level similarity
Weight Computation Process
  1. Feature Extraction: Query is analyzed for length, specificity, technical terms
  2. Corpus Sampling: ~30-40% of corpus is sampled deterministically
  3. Method Evaluation: Each method scores all samples
  4. Effectiveness Scoring: Methods scored on discrimination power & consistency
  5. Weight Conversion: Effectiveness scores → normalized weights (sum=1.0)
Key Insight: Different queries get different weights based on actual method effectiveness. Short queries boost semantic/BM25/N-gram, long queries boost field score/semantic.
